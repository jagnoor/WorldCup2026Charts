## 🧭 A guided tour (no code required)

Everything happens in your browser. Here's the full flow.

<details open>
<summary><b>🖨️ &nbsp;Build & download your wall chart</b></summary>

<br>

1. Open **[the site](https://jagnoor.github.io/WorldCup2026Charts/)** and scroll to **"Build your wall chart."**
   A control panel sits on the left; a **live preview** on the right updates the instant you change anything.
2. Set your preferences:
   - 🎨 **Colour scheme** — *Dark* shines on screen and as a poster; *Light* saves ink at home.
   - 🕑 **Time zone** — auto-detected, but pick yours. US/UK zones also show each match's TV channel.
   - 📐 **Paper size** — Letter / A3 for home printers; 24×16″ or 24×36″ for a print-shop poster.
   - ⭐ **Favourite teams** *(optional)* — tick teams to highlight them everywhere and unlock calendar export.
   - 🗣️ **Language** — English or Spanish.
3. Then use the buttons:
   - **Open poster** — full-size chart in a new tab.
   - **Print** — opens the print dialog. *Tip: Landscape, Fit-to-page, and enable Background graphics.*
   - **Download PDF / PNG** — high-res file generated right in your browser. *(Big posters take a few seconds — normal.)*
   - **Add to calendar** — pick favourite teams first, then download an `.ics` for Apple / Google / Outlook, in your local time.

</details>

<details>
<summary><b>🏆 &nbsp;Follow the knockouts live</b></summary>

<br>

Click **"Live bracket"** (top right), the hero link, or go straight to **`/knockout.html`**. This is *The Road to the Final*:

- **🌀 A survival funnel** — `48 → 32 → 16 → 8 → 4 → 2 → 1` — showing how many teams remain.
- **🗂️ Tabs for every round** — *Overview · R32 · R16 · QF · SF · Final.* Each match is a card with flags, score, your-zone kickoff, and venue.
- **🥇 Clear results** — the team that **advances** glows gold with a ▶; the one that's out is dimmed and struck through. Live matches pulse; upcoming matches count down. Played matches show **⚽ goal scorers** with the minute.
- **🛡️ A "Still standing" board** — every surviving team's flag, with the eliminated greyed out.
- **🧭 "Path to glory"** — tap any team (a chip *or* a team on a card) to **track it**: its matches glow and everything else dims, across *every* round, until you clear it.
- **↻ A Refresh button** — the page self-refreshes every few minutes, or pull the latest any time.

</details>

<details>
<summary><b>🚦 &nbsp;Reading the data banner (please do!)</b></summary>

<br>

The live bracket carries a banner that always tells you **where the numbers come from and how fresh they are** — so you're never misled:

| Badge | Meaning |
| :--- | :--- |
| 🟦 **NEAR-LIVE** | Real results from the free community feed. |
| 🟧 **SAMPLE DATA** | The feed couldn't be reached — these are *illustrative placeholders*, **not real** (and clearly labelled). |

It also shows **two timestamps** ("You last refreshed" / "Source data last updated"), spells out the limitations
in plain English, and flags any **✎ manual** result the owner hand-entered before the feed caught up.

> 🧭 **The guiding rule:** *never present a guess or a placeholder as if it were a confirmed fact.*
> Confirmed results are shown plainly; anything derived is labelled **"projected"**; anything fake is labelled **"sample."**

## 🔍 How the data stays honest

Three layers of match data, applied in order — primary first, fallback last:

```
  1. 📡  LIVE FEED        openfootball/worldcup.json — public-domain, community data.
         (primary)        Same draw as this app. Fetched in-browser, no key, no server.
                          ⤷ Near-live, not in-match live: scores land within ~a day.
              │
              ▼
  2. ✎  OVERRIDES        overrides.json — owner hand-enters a just-finished result the
         (top layer)      feed hasn't logged yet. Visibly tagged ✎ manual.
              │
              ▼
  3. 🧪  SAMPLE           results.json — bundled snapshot shown only if the feed is
         (fallback)       unreachable, so the page is never blank. Labelled "not real."
```

This project deliberately uses **free feeds only** — refresh delays are an accepted, transparent trade-off.

<details>
<summary><b>🧩 &nbsp;The bracket-resolution engine</b></summary>

<br>

The hard part of a knockout tracker: in the schedule, knockout slots aren't teams yet — they're
*placeholders.* A R32 match reads `2A vs 2B` (runners-up of groups A & B); later rounds read `W74`
(winner of match 74) or `L101` (loser of 101). `knockout.js` resolves these as results arrive:

1. **Group standings** computed via `computeStandings()` (points → goal difference → goals scored).
2. **Group slots** like `1A` / `2B` resolve to the real winner / runner-up.
3. **Third-place slots** (`3rd`) filled from the 8 best third-placed teams, avoiding a same-group rematch. *(The one genuinely heuristic step.)*
4. **Feeders** (`W74`, `L101`) cascade winners & losers forward: R32 → R16 → QF → SF → Final.
5. **Survivors** computed by removing every decided match's loser.

Each outcome is memoised per render, and scores are *oriented* to whichever side resolved — so a feed score always sticks to the correct team.

</details>

<details>
<summary><b>🛡️ &nbsp;Accuracy safeguards</b></summary>

<br>

Correctness over completeness — the app is conservative about what it claims:

- **Confirmed over guessed.** Once the feed knows a match's real teams, they override our projection.
- **"Projected" labelling.** Anything we derived but the feed hasn't confirmed is tagged **projected**; the unknown stays **TBD**.
- **A validation pass** cross-checks each result against the schedule clock and flags contradictions (e.g. a "full-time" match that can't have kicked off yet).
- **Provenance is always visible** — source, both timestamps, and limitations live in the banner; sample is labelled not-real; manual is labelled manual.

</details>

## 💻 Run it locally

No build step — any static file server works. From the project folder:

```bash
python3 -m http.server 8000        # then open http://localhost:8000
# — or —
npx serve .
```

> ⚠️ Don't double-click `index.html` — the `file://` protocol blocks the data `fetch`. Serve it over `http://`.

**Handy URL params for testing:**

| Param | Effect |
| :--- | :--- |
| `knockout.html?data=sample` | Force the bundled sample data (skip the live feed). |
| `poster.html?tz=BST&theme=light&size=A3` | Open a poster with specific settings. |

*Conventions:* assets carry a version query (`app.js?v=3`) — bump it after a deploy. No dependencies to install; `html2canvas` & `jsPDF` load from a CDN on demand, only when you click download.

## ✍️ Updating results yourself

Usually you do nothing — the feed updates on its own. But when a match **just finished** and the feed
is lagging, the owner can record it immediately by editing **`overrides.json`**:

```jsonc
{
  "updated": "2026-06-29T19:15:00Z",   // when you made this edit (UTC)
  "results": {
    // matchNumber: { home goals, away goals, status, team codes }
    "76": { "h": 2, "a": 1, "status": "FT", "_t1": "BRA", "_t2": "JPN" }
  }
}
```

- `h` / `a` → **home / away** goals · `_t1` / `_t2` → home / away **team codes** (3-letter, from `schedule.js`).
- `status` is usually `"FT"`. For a shootout add `"pen": [homePens, awayPens]`.
- The match number is the `#` on each card (R32 is 73–88, the Final is 104).

Save, hit **↻ Refresh data** — it appears tagged **✎ manual** and cascades through the bracket.
**Remove the entry** once the live feed catches up, so the feed becomes the source of truth again.

It's static, so **GitHub Pages** hosts it for free:

2. **Settings → Pages** → set the source to the `main` branch (root).

## ⚠️ Accuracy, limitations & disclaimers

- **Not an official FIFA product.** A fan-made tool — for authoritative scores always check the official FIFA app or your broadcaster.
- **Near-live, not in-match live.** The free feed updates roughly daily; results can lag by hours. Shown clearly in the UI; a deliberate, cost-free trade-off.
- **Third-place pairings are a best-effort projection** until the feed confirms the real R32 matchups.
- **Group tie-breakers** apply the official FIFA order: points → goal difference → goals scored, then a head-to-head mini-league. Fair-play points and drawing of lots are approximated by the original seeding.
- **Sample data is fictional** — fallback scores are illustrative placeholders, labelled as such.

See **[`BACKLOG.md`](BACKLOG.md)** for the roadmap and known limitations.
